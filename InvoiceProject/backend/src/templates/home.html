Invoice Rendering API

By DONUT

This is a quick and simple guide on how to use this API on Postman. More detail can be found on SwaggerHub Documentation.


All Routes Except auth/register and healthcheck:

  • Users will need to send the request with an authorization header with the key “x-api-key” and its value set to their personal API key.

render/html:

  • Users will send the POST request with body containing form-data which will have the key “xml_file” and its value set to their input XML file.
  • Users can view the response headers to retrieve the render_id of the rendered invoice.
  • RETURNS: html file

render/delete:

  • Users will send the DELETE request with params containing the key “render_id” with the value of their render_id.
  • RETURNS: {}

render/view:

  • Users will send the GET request with params containing the key “render_id” with the value of their render_id
  • RETURNS: {'url': url} (download link of rendered invoice)

auth/register:

  • Users will send the POST request with body containing form-data which will have the key “email” with the value of their email string and the key “password” with the value of their password string.
  • Password must have at least 5 chars
  • Register will not work if the email has already been registered
  • RETURNS: {'api_key': api_key}

auth/login:

  • Users will send the POST request with body containing form-data which will have the key “email” with the value of their email string and the key “password” with the value of their password string.
  • RETURNS: {'api_key': api_key}

healthcheck:

  • RETURNS: {"success": true} if server is functioning.
